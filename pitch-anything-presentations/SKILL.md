# Create presentations from theme + description

Treat every presentation as a cognitive sequence, not a data dump.

Use this skill to:
- ask the right discovery questions first,
- define the presentation strategy,
- structure the deck,
- assign persuasive framing,
- produce slide-by-slide guidance,
- close with a clear decision path.

Base the presentation on these principles:
- frame first
- narrative before analysis
- hot cognition before cold cognition
- clarity before detail
- decision before discussion drift

## Workflow

### 1. Run discovery first
Do not assume critical context.

If audience, objective, time, meeting type, resistance, or desired output are missing, ask the necessary questions first.

Read:
- `references/discovery.md`

### 2. Define the presentation strategy
After discovery, define:
- audience
- desired outcome
- core tension
- opening frame
- emotional objective
- rational objective

If needed, adapt the strategy to the context.

Read:
- `references/frames.md`
- `references/adaptation-by-context.md`

### 3. Build the deck arc
Choose the right deck length and structure the sequence of slides.

Default arc:
- reframe the room
- put a human in the problem
- explain why now
- reveal the insight
- show the solution
- show the prize
- provide proof
- ask for decision

Read:
- `references/deck-arc.md`

### 4. Write the output
For each slide include:
- slide title
- goal
- what to say
- what to show visually
- cognitive job / frame
- transition notes

### 5. Validate before finalizing
Check whether the presentation:
- opens with tension or curiosity,
- delays heavy detail until interest exists,
- addresses likely resistance,
- has a clear why-now,
- ends with a real ask.

Read:
- `references/quality-check.md`

## Output modes

### Quick outline
Return:
- strategy
- 6–8 slide titles
- one sentence per slide

### Full deck brief
Return:
- strategy
- deck length
- slide-by-slide structure
- visual suggestions
- objections and responses
- CTA

### Speaker notes
Expand each slide with:
- intention
- message
- transition
- optional phrasing

## Rules
- Ask necessary questions before drafting if context is incomplete.
- Do not open with dense explanation unless the user explicitly wants a technical deep dive.
- Do not dump features before relevance is established.
- Do not place proof too early.
- Do not end without a concrete next step.
